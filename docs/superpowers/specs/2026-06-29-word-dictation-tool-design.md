# 儿童单词听写助手 · 设计文档

- 日期：2026-06-29
- 形态：本地优先的 PWA（渐进式 Web 应用）
- 目标用户：需要练习单词/词语听写的儿童及陪练家长

## 1. 概述与目标

一个帮助儿童背诵、听写单词的工具。家长创建单词本（英文或中文），App 用语音合成（TTS）逐个朗读，孩子在**纸上手写**作答，听写完后逐条对照标记对错；标错的词进入"错词本"可重练，所有词依据**简化遗忘曲线**自动安排下次复习日期，首页汇总"今日待复习"。英文词条自动联网获取并显示**音标**。

核心价值：
1. 真实还原学校听写流程（听音 → 纸上默写 → 对照批改）。
2. 基于间隔重复的科学复习，自动安排错词与到期词的再听写。
3. 纯本地优先、免账号、可离线、可"添加到主屏幕"，并支持数据备份。

## 2. 技术选型

- **方案**：原生 JavaScript（不使用 React/Vue 等框架）+ Vite 开发/打包 + `vite-plugin-pwa`。
- **存储**：浏览器 IndexedDB（结构化数据），少量 UI 偏好存 localStorage。
- **朗读**：浏览器 Web Speech API（`speechSynthesis`），中文用 `zh-CN`、英文用 `en-US`。
- **音标获取**：联网调用免费在线词典 API（Free Dictionary API，`https://api.dictionaryapi.dev/api/v2/entries/en/<word>`，免费、无需密钥）。**仅在导入 / 手动添加英文词时调用一次**，把音标文本缓存进本地词条；日常听写与复习无需联网。
- **运行环境**：已确认本机安装 Node.js v24.14.0 / npm 11.12.0。
- 无后端服务、无用户账号；首次加载后核心功能可离线，仅"录入英文词取音标"这一步需要网络。

理由：项目规模适中（约 5 个页面），原生 JS 代码轻、易懂、易维护；Vite + PWA 插件让离线缓存与"加主屏"几乎零配置。

## 3. 页面与导航

底部导航三个入口：**首页 / 单词本 / 错词本**。

1. **首页 / 今日复习**
   - 顶部："今日待复习 N 个词"，一键开始复习听写（跨单词本汇总到期词）。
   - 下方：单词本列表入口、空状态引导。
2. **单词本管理**
   - 新建单词本（选类型：英文 / 中文）。
   - 查看 / 编辑 / 删除单词本；本内：批量粘贴导入、手动添加、词条列表（英文条目显示音标）。
3. **听写设置页**（开始某次听写前）
   - 选范围：整本 / 勾选部分词 / 今日复习 / 错词本。
   - 英文本选模式：读英文写英文 / 读中文写英文。
   - 朗读控制：手动 or 自动播放、自动间隔秒数（5–20，默认 8）、可重复朗读。
4. **听写进行页**
   - 大字进度（第 3 / 20）、"再读一遍"、"下一个"；自动播放可暂停。
   - **不显示任何答案文字（含音标）**，避免孩子看到拼写。
5. **核对 / 批改页**
   - 逐条列出正确答案（英文/汉字 + 音标 + 释义 + 喇叭重听）。
   - 家长逐条标记 ✓ / ✗。
   - 完成后：错词置入错词本，并按复习算法更新每个词的下次复习日期。
   - "立即重练错词"按钮。

## 4. 数据结构（IndexedDB）

两张对象存储：`wordbooks`、`words`。无历史记录表，复习调度仅依赖词条自身状态。

**Wordbook**
```
id          唯一ID
name        名称（如"五年级上 Unit 3"）
type        'en' 英文 | 'zh' 中文
createdAt   创建时间
```

**Word**
```
id          唯一ID
wordbookId  所属单词本
text        主词：英文拼写 或 中文词语
meaning     中文释义（英文本必填；中文本选填）
phonetic    音标（仅英文本，联网自动获取并缓存，可手动编辑，允许为空）
pinyin      拼音（仅中文本，选填）

—— 复习状态 ——
level       间隔档位索引（0,1,2…）
dueDate     下次复习日期（到这天进入"今日待复习"）
isWrong     是否在错词本（被标错 → true）
lastReviewedAt  上次听写日期
createdAt
```

`words` 建立索引：`wordbookId`、`dueDate`、`isWrong`，便于查询今日到期词与错词。

## 5. 复习算法（简化遗忘曲线）

间隔档位天数：`[1, 2, 4, 7, 15, 30]`

- **新词导入后**：`level = 0`，`dueDate = 今天`（当天可首次听写）。
- **标 ✓ 对**：`level = min(level + 1, 5)`；`dueDate = 今天 + 档位天数[新level]`；若原在错词本则 `isWrong = false`。
- **标 ✗ 错**：`level = 0`；`dueDate = 明天`；`isWrong = true`。
- `lastReviewedAt = 今天`。

派生集合：
- **今日待复习** = 所有 `dueDate ≤ 今天` 的词，跨单词本汇总。
- **错词本** = 所有 `isWrong = true` 的词。

日期按本地日期（仅到"天"，忽略时分秒）比较。

## 6. TTS 朗读对应关系

- 英文本「读英文写英文」→ 朗读 `text`，语音 `en-US`。
- 英文本「读中文写英文」→ 朗读 `meaning`，语音 `zh-CN`。
- 中文本 → 朗读 `text`，语音 `zh-CN`。

进入 App 时检测可用语音；若设备缺中文或英文语音，给出提示（个别手机需在系统设置安装语音包）。首次朗读需由用户点击触发（满足浏览器自动播放限制）。

## 7. 音标获取（英文词条）

- **时机**：仅在批量导入或手动添加英文词时，对每个词调用一次 Free Dictionary API 取音标文本，写入 `phonetic` 字段缓存；此后离线显示。
- **接口**：`GET https://api.dictionaryapi.dev/api/v2/entries/en/<word>`，从返回中取 `phonetic` 或 `phonetics[].text`（取第一个非空者）。
- **失败 / 离线降级**：查不到、超时或无网络时 `phonetic` 留空，不阻塞导入；词条列表里可对留空项一键"重新获取"，也可手动填写。
- **批量节流**：导入多词时对请求做适度并发限制（如每次 3–5 个），失败项跳过并计数提示。
- **显示位置**：词条列表、核对页正确答案处显示音标；听写进行页不显示。

## 8. 批量导入格式

每行一个词，自动识别分隔符（制表符 / 英文逗号 / 中文逗号 / 连续空格），方便从课本单词表或 Excel 直接粘贴。

- **英文本**：第一段 = 英文，其余 = 中文释义（音标无需手填，导入后自动联网获取）。
  ```
  apple    苹果
  banana，香蕉
  go to school 去上学
  ```
- **中文本**：第一段 = 词语，第二段（可选）= 拼音。
  ```
  葡萄
  美丽  měi lì
  ```

导入流程：粘贴 → **预览解析结果**（显示将导入 N 条、每条拆分情况）→ 确认入库 → 后台为英文词获取音标。空行跳过；本内重复词给出提示。

## 9. 数据备份（导出 / 导入）

- **导出**：一键将全部单词本 + 词条（含复习进度与音标）导出为单个 `.json` 文件，保存到设备。
- **导入**：选择备份 `.json` 恢复数据；导入时提示是"合并"还是"覆盖现有数据"（默认覆盖，给出明确确认）。
- 目的：纯本地存储下，换设备或清理浏览器缓存会丢数据，备份用于兜底。

## 10. 关键交互细节

- 听写进行页全程不显示答案文字（含音标）。
- 自动播放默认间隔 8 秒，可在设置中调（5–20 秒）。
- 空状态：无单词本时首页引导"先新建一个单词本"。
- 适配手机/平板触屏，按钮大、字号大，适合儿童与家长操作。

## 11. 非目标（YAGNI）

- 不做用户账号、云同步、多人协作。
- 不做屏幕手写识别、不做打字自动批改（明确采用纸上手写 + 人工标记）。
- 不做听写历史统计报表（首期仅保留每词当前复习状态）。
- 朗读不接入在线发音音频，仅用设备本地 TTS（在线词典仅用于获取音标文本）。
- 复习算法不采用 Anki SM-2 动态算法，仅用固定档位。

## 12. 测试要点

- 复习算法：新词初始状态、答对升档与到顶封顶、答错退回明天、错词进/出错词本；日期按"天"比较的边界。
- 导入解析：各分隔符、英文/中文本、空行、重复词、缺释义/拼音。
- 音标获取：成功取值、查不到 / 超时 / 离线降级为空、批量节流、重新获取（接口用 mock 验证逻辑）。
- 数据层：IndexedDB 增删改查、索引查询（今日到期、错词）。
- 备份：导出后导入的往返一致性、覆盖 vs 合并。
- TTS：语音选择与缺失语音的降级提示（可用 mock 验证逻辑）。
