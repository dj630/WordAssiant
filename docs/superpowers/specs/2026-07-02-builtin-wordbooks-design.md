# 内置单词本（人教版2024 · 三/四年级）· 设计文档

- 日期：2026-07-02
- 依赖：已完成的"单词听写助手" PWA（见 `2026-06-29-word-dictation-tool-design.md`）
- 目标：随 App 内置人教版 2024 新版小学英语词表，用户可一键把某单元添加进自己的词库并直接听写/复习。

## 1. 概述与目标

在现有 App 中加入**内置词库**：预置人教版 2024 新版小学英语**三年级、四年级**的单词/短语，**一个单元一个单词本**。用户在"单词本"页进入内置词库目录，点"添加到我的词库"即把该单元存成一本普通单词本，之后听写、核对、遗忘曲线复习、错词本等全部功能照常可用。

数据 100% 来自用户提供的官方词表文件（`docs/pep2024-g3-g4-wordlist.txt`），不由模型臆造，保证准确。

范围仅三、四年级（2024 新版当前只启用到这两个年级）；数据结构预留扩展，5/6 年级教材发布后可追加。

## 2. 数据规模（已解析校验）

源文件解析得 **24 本、486 词条，0 处解析错位**：

| 册 | 单元数 | 词条合计 |
|----|--------|----------|
| 三年级上册 | 6（U1–U6） | 109 |
| 三年级下册 | 6 | 90 |
| 四年级上册 | 6 | 101 |
| 四年级下册 | 6 | 186 |

每单元含单词与短语（如 `go to school 去上学`、`look after 照顾`）；**词性已写在中文释义里**（`n. 苹果`、`v./n. 微笑；笑`、`adj. 高兴的`）。短语通常无词性。

## 3. 数据来源与生成

- 源词表：`docs/pep2024-g3-g4-wordlist.txt`（由项目根目录的 `.txt` 移入留档，格式为"英文一行、中文一行"，夹带表头/网址噪音行）。
- **一次性生成脚本** `scripts/gen-builtin-books.mjs`：解析源文件，产出静态数据模块 `src/data/builtin-books.js` 并提交入库。运行时**零网络、零解析**，直接 `import`。
- 解析规则：识别 `X年级上/下册（2024新版）` 为册、`Unit N Title（中文）` 为单元；跳过噪音行 `英文` / `中文释义` / `￼` / `book118.com` / `docin.com` / `xuehi.cn` / `zxxk.com` / `+数字`；其余按"英文行 + 中文行"成对为词条。

**`src/data/builtin-books.js` 结构**：导出 `BUILTIN_BOOKS` 数组，每项：

```
{
  id: 'pep2024-g3-t1-u1',        // 稳定唯一串：年级g3/g4、册t1(上)/t2(下)、单元uN
  name: '三上 U1 Making Friends', // 简短本名（见 §4）
  type: 'en',                     // 均为英文本
  grade: '三年级',                // 年级属性（见 §5）
  term: '上册',                   // 册（用于分组与命名）
  unit: 1,
  title: 'Making Friends',        // 英文单元名
  unitZh: '交朋友',               // 中文单元名（目录页展示）
  entries: [ { text: 'nice', meaning: 'adj. 令人愉快的；友好的' }, ... ],
}
```

## 4. 单词本命名

简短格式：`<年级简写><册简写> U<单元> <英文单元名>`，例：

- `三上 U1 Making Friends`
- `四下 U6 From Farm to Table`

年级简写：三年级→`三`、四年级→`四`；册简写：上册→`上`、下册→`下`。中文单元名（`交朋友`）不进本名，仅在内置词库目录页作为副标题展示。

## 5. 单词本"年级"属性与筛选

### 5.1 数据模型变更（向后兼容）
`Wordbook` 增加可选字段 `grade`（字符串），存**"年级+册"组合值以区分上下册**：
- 内置本添加时写入 `'三年级上册'` / `'三年级下册'` / `'四年级上册'` / `'四年级下册'`（由内置数据的 `grade + term` 拼成）。
- 手动新建本：表单新增可选"年级"下拉（`不限` / `三年级上册` / `三年级下册` / `四年级上册` / `四年级下册`），默认 `不限`（存为 `''`）。
- 既有本（升级前创建、无该字段）读取时视为 `''`（未分类）。IndexedDB 按记录存储，无需迁移，`DB_VERSION` 不变。

`createWordbook` 接受并存储 `grade`（缺省 `''`）。

### 5.2 "单词本"页年级筛选
"单词本"列表页顶部加一排筛选块：`全部` + 数据中出现的各"年级+册"分类（如 `三年级上册`/`三年级下册`/…，天然区分上下册）+ （存在未分类本时显示）`未分类`。选中后仅显示该分类的本。筛选选项**由当前 `getWordbooks()` 结果动态派生**，避免硬编码。默认 `全部`。

## 6. 界面与接入

不新增底部导航项（保持 首页/单词本/错词本 三项）。

### 6.1 "单词本"页（`src/pages/wordbooks.js`，改）
- 顶部加入口按钮 **「＋ 内置词库（人教版2024）」** → `navigate('#/builtin')`。
- 加 §5.2 年级筛选（区分上下册）。
- 列表项在本名与类型旁，若有 `grade` 则以 `muted` 小字显示"年级+册"。
- 新建表单加可选"年级"下拉。

### 6.2 内置词库目录页（`src/pages/builtin.js`，新增）
- `renderBuiltin(): Promise<HTMLElement>`。
- 从 `BUILTIN_BOOKS` 按 `grade + term` 分组（三上/三下/四上/四下），每组标题下逐单元一行：`本名 + 中文单元名 + 词数 + 【添加到我的词库】`。
- 已存在同名用户单词本时，按钮旁显示 `已添加` 提示，但**仍允许再次添加**（用户可能删除过）。
- 底部 `renderNav('wordbooks')`。
- 路由：`src/router.js` 增 `{ name: 'builtin', pattern: /^#\/builtin$/, keys: [] }`；`src/main.js` 注册 `builtin: renderBuiltin`。

### 6.3 "添加到我的词库"逻辑
点击某单元的添加按钮：
1. `createWordbook({ name, type: 'en', grade: book.grade + book.term })`（即 `三年级上册` 之类的组合值）。
2. `addWords(wb.id, entries, todayStr())` 写入全部词条（`meaning` 即"词性+释义"）。
3. 英文音标回填：复用现有 `fetchPhoneticsBatch` + `updateWord`（与手动导入一致）；短语查不到音标则留空。回填期间按钮显示"添加中…"，完成显示"已添加 ✓"。
4. 不自动跳转，便于用户连续添加多个单元。

## 7. 测试要点

- **数据模块** `src/data/builtin-books.test.js`：`BUILTIN_BOOKS` 恰 24 本；`id` 唯一；每本 `entries` 非空且每条含非空 `text`/`meaning`；`grade` ∈ {三年级, 四年级}、`term` ∈ {上册, 下册}；`type` 均为 `en`；总词条数 486。
- **添加逻辑** `src/pages/builtin.test.js`（jsdom + fake-indexeddb，音标 fetch 用 mock）：点某单元"添加"后，`getWordbooks()` 多一本且 `grade` 为组合值（如 `三年级上册`），`getWordsByBook` 返回该单元全部词条。
- **上下册筛选** `src/pages/wordbooks.test.js`（扩充）：建三上、三下、四上各一本 + 一本未分类，选 `三年级上册` 后列表只剩该分类本（验证区分上下册）。
- **命名/分组**：目录页渲染出 4 个分组标题与正确本名（抽查 `三上 U1 Making Friends`）。
- 生成脚本本身不纳入单元测试（一次性构建工具）；其正确性由数据模块测试与人工抽查保证。

## 8. 非目标（YAGNI）

- 不做内置本的在线更新/版本管理（词表随代码发版）。
- 不做"添加时按名去重/合并"（仅提示"已添加"，仍允许重复添加）。
- 暂不含 5/6 年级（教材未发布）。
- 不为内置短语强行造音标（单词词典 API 查不到即留空）。
- 不预置音标到数据文件（沿用导入时联网回填并缓存的既有策略）。
